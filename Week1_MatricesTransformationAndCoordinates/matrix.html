1. Matrix(행렬)에 대해 알아두어야 할것
    • 위키피디아 참고
    • 이것만은 기억하자
      • 복수형은 matrices
      • 표기법
      • 수학적 성질
        • 기본 연산(덧셈, 뺄셈, 곱셈, 그리고 나눗셈?)
        • 단위행열(identity matrix)은 곱셈에 대한 항등원 - 우상단 부터 좌하단까지 1이고 나머지는 0인 행과 열의 수가 같은 정사각(square) 행열
        • 역행열(inverse matrix)는 곱셈에 대한 역원
        • 곱셈에 대한 교환법칙은 성립하지 않음 (AB != BA)
  2. GL에서의 행열과 좌표계(coordinate system)
    • 참고 사이트
    • 4x4 크기의 정사각 column major matrix 사용 / 오른손 좌표계를 사용
    • 열 단위로 저장, 벡터를 곱할때는 열 벡터 사용 / 오른손으로 좌표계의 x, y, z축의 방향을 알 수 있다
    • 행열과 변환 이동(transform) 그리고 투영(projection)
      • 우상단 3x3 부분이 rotation(회전)
      • 최우측 열의 3x1 벡터가 x, y, z translate(이동)
      • a1,1, a2,2, a3,3은 x, y, z scale(배율)
      • 점의 변환 이동은 보통 TRS(translate, rotate, scale) 순으로 적용
      • 하지만 T x R x S 가 아니라 S x R x T 임을 명심하자
      • 점의 화면 렌더 위치 계산법 카메라 MVP
        • (M)odel: 객체의 변환이동 정보(4x4 행열) x 점의 좌표(열벡터)
        • (V)iew: 카메라 위치 기준으로 model의 변환 이동 정보로 바꿔주는 정보(look at - 4x4 행열)
        • (P)rojection: 카메라의 투영법(4x4 행열)
        • 여기서도 역시 P x V x M 임을 명심하자
